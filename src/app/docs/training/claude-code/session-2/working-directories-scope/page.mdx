# Working Directories & Scope

<ReadTime minutes={2} />

Claude Code operates from wherever you launch it.

1. **Your current directory is Claude's world** — it won't look outside unless you tell it to
2. **`cd` into the right repo before running `claude`** — this scopes all operations
3. **Use separate sessions for separate repos** — don't mix infrastructure and identity work
4. **Think of each session as a focused workspace,** not a general-purpose assistant

<br />

<Callout variant="info" title="Same rule as context windows" align="left">
  **Scope tight, work focused, start fresh between tasks.** A session that
  jumps across three repos and four problem domains will perform worse than
  three short focused sessions.
</Callout>
