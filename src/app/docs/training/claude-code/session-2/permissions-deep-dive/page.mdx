## Notes

**Start strict, relax selectively.** Begin with **Ask Every Time** for unfamiliar repos. As you learn which commands and edits are safe, add `allow` rules for those specific patterns. Never start permissive and try to lock down later.

<br />

<Callout variant="error" title="Deny rules are absolute">
  A `deny` rule **always wins**, regardless of where it's defined. This makes
  guardrails reliable: a project-level deny cannot be overridden by a more
  permissive user setting.
</Callout>
