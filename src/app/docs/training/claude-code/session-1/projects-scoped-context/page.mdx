# Projects: Scoped Context

<ReadTime minutes={2} />

Give Claude persistent instructions and reference documents for a specific area of work.

## What Goes in a Project

- **Custom instructions** — rules Claude follows for every conversation in this project
- **Reference documents** — upload style guides, architecture docs, runbooks, policies
- **Scoped memory** — Claude's memory stays focused on this project's context
- **Team knowledge** — acronyms, conventions, tool preferences, compliance rules

## Example Projects

| Project | Contents |
|---------|----------|
| **IAM Governance** | CA policy templates, Graph SDK conventions, compliance rules |
| **Infrastructure** | Terraform modules, naming standards, deployment procedures |
| **Incident Response** | Runbook templates, escalation matrix, KQL query library |
| **Team Onboarding** | Architecture docs, coding standards, tool setup guides |

---

> *Create a project for each major work area. Instructions + reference docs = Claude that knows your context from the first message.*
