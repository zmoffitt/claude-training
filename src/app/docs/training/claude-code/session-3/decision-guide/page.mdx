# Skill, Routine, MCP, or Plugin?

<ReadTime minutes={2} />

<div className="max-w-2xl lg:max-w-3xl">

| When you need to… | Skill | Routine | MCP / Connector | Plugin |
| --- | :---: | :---: | :---: | :---: |
| Encode a team workflow Claude should follow | **Best fit** | — | — | — |
| Pull data from an approved SaaS (M365, GitHub) | — | — | **Built-in connector** | — |
| Run Claude unattended on a schedule or event | — | **Best fit** | often uses | — |
| Distribute several capabilities to the whole team | — | — | — | **Best fit** |
| Standardize how a runbook gets executed | **Best fit** | often pairs | often pairs | often pairs |

</div>

A **routine** runs a **skill** that uses a **connector**, and the whole thing can ship inside a **plugin**. The interesting decision is usually _which connector / MCP_, and _does it need to run while I'm away_ (routine) or with me at the keyboard (skill).
