# Plugins & Marketplaces

<ReadTime minutes={2} />

## What's in a plugin

- One or more **skills** (`SKILL.md` files)
- Optional **subagents** with focused prompts
- Optional **MCP server** config
- Optional **hooks** and slash commands
- Versioned, distributed via marketplaces

## Install flow

```bash
# Add a marketplace
/plugin marketplace add org/plugins

# Browse and install
/plugin install iam-tools@org-plugins

# List what's installed
/plugin list
```

<br />

<Callout variant="info" title="Private marketplace = team standardization" align="left">
  Build a single internal plugin repo. Every engineer installs it. New
  conventions, skills, or MCP servers ship through one update — no
  copy-paste, no drift.
</Callout>
