# MCP Architecture

<ReadTime minutes={3} />

A protocol — not a product. Open standard, any client, any server.

Claude is the **client**. An **MCP server** exposes tools and data over JSON-RPC, running over stdio (local) or HTTP/SSE (remote). The server brokers calls to an external system (Microsoft Graph, GitHub, Kubernetes, a database) and returns results. You stay in the loop, approving or denying each tool call.

## Three primitives

- **Tools** — functions Claude can call (`create_ticket`, `list_users`)
- **Resources** — data Claude can read (logs, dashboards, docs)
- **Prompts** — pre-built prompt templates the server offers

## Local vs. remote

- **Remote-first (HTTP/SSE)** reaches web, mobile, and managed agents
- **Local (stdio)** is the developer-loop case: quick to iterate, single user

The same protocol works in Claude Desktop, Claude Code CLI, and the web — write a server once, use it everywhere.
